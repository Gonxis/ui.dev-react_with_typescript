    * This is because the type being returned by useHover() is `[boolean | {onMouseOut: ()=>void, onMouseOver: ()=>void}]`
    We need to tell ts that the return of useHover is a tuple, not an arbitrary array of booleans and objects. I did this by creating a `MouseEventsInterface` that has the 2 event types, and then specified in the `userHover()` declaration that the return type is `[boolean, MouseEventsInterface]`
    Here is my useHover code that is not giving an error with ts 4.9.3

    ```ts
    import React from 'react'

    export interface MouseEventsInterface {
        onMouseOver: ()=>void
        onMouseOut: ()=>void
    }


    export default function useHover ():[boolean, MouseEventsInterface] {
        const [hovering, setHovering] = React.useState(false)

        const onMouseOver = () => setHovering(true)
        const onMouseOut = () => setHovering(false)

        return [hovering, {
            onMouseOut,
            onMouseOver
        }]
    }
    ```
